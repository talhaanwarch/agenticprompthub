Phase 1 — In Scope vs. Deferred

🔐 Auth & Teams

  • Email / password auth (JWT sessions)
  • Team creation and invite-by-email flow
  • Roles: Owner, Admin, Member, Viewer
  • team_id on every row (RLS-ready for org layer later)

📝 Prompt CRUD

  • Create / edit prompts with name, description, tags
  • Variable interpolation with {{variable}} syntax
  • Slug-based identifiers for SDK fetching
  • Soft-delete with audit trail entries

🔖 Versioning

  • Immutable versions — published versions never mutate
  • Movable environment aliases: production, staging, latest
  • Version history with author + timestamp
  • Promote / rollback alias without touching version record

🔍 Compare & SDK

  • Side-by-side text diff between any two versions
  • REST API for runtime prompt fetch by slug + env alias
  • JavaScript / TypeScript SDK (Phase 1 target)
  • Export / sync as secondary distribution path

Explicitly deferred to later phases

AI Proxy / BYOK Playground / Live Test Conversation Traces Evaluation / Datasets LLM-as-Judge Org Hierarchy SSO / SAML Multi-agent Workflows Output Comparison

Foundational Design Decisions

🏗️

Tenancy Model

Team-scoped MVP with team_id on every resource and Postgres Row-Level Security from day one. Org hierarchy added later without restructuring.

Team-first, org-ready
🔖

Versioning Strategy

Immutable published versions with movable environment alias pointers. Apps pin to an alias, not a version number — rollback = move the alias.

Aliases over pinned IDs
🔍

Prompt Comparison

Phase 1 scope is text diff only (what changed). Output comparison and full eval belong to later phases to keep Phase 1 self-contained.

Text diff only in P1
📡

Distribution Model

Runtime SDK fetch is the primary path (creates stickiness). Export/sync is secondary. Apps never hardcode prompt text.

SDK-first runtime fetch
📦

Open Source Path

Architecture keeps the self-hostable path open even though the OSS/commercial decision is deferred. No SaaS-only coupling in Phase 1.

Open-core compatible
🐘

Backend Language

Express/TypeScript for Phases 1–2. Python integration considered for Phases 4–5 where the AI/eval ecosystem strongly prefers Python.

TS now, Python later
