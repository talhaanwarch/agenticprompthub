# Backend Part 4 — TypeScript SDK: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `@agentichub/sdk` — a thin TypeScript package that lets Node.js apps call `renderPrompt(name, alias, variables)` and receive an OpenAI-compatible messages array, backed by a stale-while-revalidate LRU cache and configurable retry logic.

**Architecture:** Single class `AgenticHub` with a module-level LRU cache singleton. All Jinja2 rendering happens server-side; the SDK sends variables to `POST /prompts/:name/:alias/render` and returns the result. Config resolves from constructor → env var → default. Fail-fast constructor validation. Background refresh on stale cache hits.

**Tech Stack:** TypeScript 5.x, Node.js 18+ (native `fetch`), `lru-cache`, `tsup` for dual CJS+ESM output, `vitest` for unit tests, `supertest` for integration tests.

## Global Constraints

- TypeScript strict mode (`"strict": true`). No `any` without a comment explaining why.
- No plain `.js` files in `src/`. All source files are `.ts`.
- Node.js 18+ required — uses native `fetch`. No polyfill, no `node-fetch`.
- No nunjucks dependency — all Jinja2 rendering is server-side.
- No browser support — defer to a future phase.
- Module-level cache singleton: `maxCacheSize` is set by the first constructor call.
- Variables are NOT part of the cache key.
- Integration tests: real Postgres + real Express app (via supertest), no mocks for our own API.
- External paid API calls (e.g. LiteLLM) are the only acceptable mock target — not applicable in B4.
- JSDoc on every exported symbol.
- Every domain folder (here: `packages/sdk/src/`) uses an `index.ts` barrel.
- No commits without explicit user permission.

 * All fields are optional; constructor falls back to environment variables,
 * then to built-in defaults. `apiKey` and `baseUrl` throw if neither is provided.

export interface AgenticHubConfig {
  /** API key for authentication. Fallback: process.env.AGENTICHUB_API_KEY */
  apiKey?: string;
  /** Base URL of the AgenticHub API. Fallback: process.env.AGENTICHUB_BASE_URL */
  baseUrl?: string;

  maxCacheSize?: number;

  maxRetries?: number;
  /** Milliseconds to wait between retries. Default: 500. */
  retryInterval?: number;

```typescript
import { LRUCache } from 'lru-cache';
import type { Message } from './types';

/** Shape of each entry stored in the LRU cache. */
export interface CacheEntry {
  /** The rendered messages array returned by the API. */
  value: Message[];
  /** Unix timestamp (ms) when this entry was fetched. Used to compute age for SWR. */
  fetchedAt: number;
}

/**
 * Module-level LRU cache singleton.
 * Initialised on the first getCache() call; subsequent calls return the same instance.
 * Using a module-level singleton (rather than lru-cache's built-in TTL) so that
 * stale entries can be served while a background refresh is in flight —
 * lru-cache's TTL evicts on read, which would force a blocking fetch on every stale hit.
 */
let _cache: LRUCache<string, CacheEntry> | null = null;

/**
 * Returns the module-level LRU cache, creating it on the first call.
 * `maxCacheSize` is only honoured on the first call; subsequent calls return
 * the already-initialised cache regardless of the argument.
 *
 * @param maxCacheSize - Maximum number of entries. Effective only on the first call.
 * @returns The shared LRU cache instance.
 */
export function getCache(maxCacheSize: number): LRUCache<string, CacheEntry> {
  if (!_cache) {
    _cache = new LRUCache<string, CacheEntry>({ max: maxCacheSize });
  }
  return _cache;
}

/**
 * Resets the module-level cache singleton to null.
 * FOR TESTING ONLY — do not call in production code.
 * @internal
 */
export function _resetCacheForTesting(): void {
  _cache = null;
}
```

```typescript
/**
 * Pauses execution for the given number of milliseconds.
 *
 * @param ms - Duration to sleep in milliseconds.
 */
export async function sleep(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

/**
 * Wraps the global `fetch` with retry logic for transient failures.
 *
 * Retries on:
 *   - Network-level errors (fetch() throws)
 *   - HTTP 5xx responses
 *
 * Does NOT retry on HTTP 4xx — those are client errors; retrying won't help.
 *
 * Total attempts = 1 + maxRetries (e.g. maxRetries=1 → 2 total attempts).
 *
 * @param url - The URL to fetch.
 * @param init - The RequestInit options to pass to fetch.
 * @param maxRetries - Number of additional attempts after the first failure.
 * @param retryInterval - Fixed delay in ms between retry attempts.
 * @returns The Response from the last successful attempt.
 * @throws The last network error if all attempts fail with a network error.
 */
export async function fetchWithRetry(
  url: string,
  init: RequestInit,
  maxRetries: number,
  retryInterval: number,
): Promise<Response> {
  let lastError: unknown;
  const totalAttempts = 1 + maxRetries;

  for (let attempt = 0; attempt < totalAttempts; attempt++) {
    if (attempt > 0) {
      await sleep(retryInterval);
    }

    try {
      const response = await fetch(url, init);

      // 4xx: do not retry — return immediately for caller to handle
      if (response.status >= 400 && response.status < 500) {
        return response;
      }

      // 5xx: retry if we have attempts left
      if (response.status >= 500) {
        lastError = new Error(`HTTP ${response.status}`);
        if (attempt < totalAttempts - 1) {
          continue;
        }
        // All attempts exhausted — return the last 5xx response so caller can inspect body
        return response;
      }

      // 2xx/3xx: success
      return response;
    } catch (err) {
      // Network-level error (DNS failure, connection refused, etc.)
      lastError = err;
      if (attempt < totalAttempts - 1) {
        continue;
      }
      throw err;
    }
  }

  // TypeScript: this is unreachable but satisfies exhaustiveness
  throw lastError;
}

```typescript
import type { Message, AgenticHubConfig } from './types';
import { AgenticHubError } from './error';
import { getCache } from './cache';
import { fetchWithRetry } from './fetch';

const DEFAULT_CACHE_TTL = 60_000;
const DEFAULT_MAX_CACHE_SIZE = 500;
const DEFAULT_MAX_RETRIES = 1;
const DEFAULT_RETRY_INTERVAL = 500;

/**
 * Client for fetching rendered prompts from AgenticHub.
 *
 * Create one instance at process startup and reuse it — the cache is a
 * module-level singleton and `maxCacheSize` is set by the first constructor call.
 *
 * @example
 * ```typescript
 * const hub = new AgenticHub({ apiKey: process.env.AGENTICHUB_API_KEY, baseUrl: process.env.AGENTICHUB_BASE_URL });
 * const messages = await hub.renderPrompt('my-prompt', 'production', { name: 'Alice' });
 * ```
 */
export class AgenticHub {
  private readonly apiKey: string;
  private readonly baseUrl: string;
  private readonly cacheTtl: number;
  private readonly maxRetries: number;
  private readonly retryInterval: number;

  /**
   * Initialises the AgenticHub client. Throws immediately on missing required config.
   *
   * Config resolution order for each field: constructor arg → environment variable → default.
   *
   * @param config - Optional configuration overrides.
   * @throws {AgenticHubError} MISSING_API_KEY if no apiKey is provided and AGENTICHUB_API_KEY is unset.
   * @throws {AgenticHubError} MISSING_BASE_URL if no baseUrl is provided and AGENTICHUB_BASE_URL is unset.
   */
  constructor(config?: AgenticHubConfig) {
    const apiKey = config?.apiKey ?? process.env.AGENTICHUB_API_KEY;
    if (!apiKey) {
      throw new AgenticHubError(
        'AgenticHub: apiKey is required. Pass it in the constructor or set AGENTICHUB_API_KEY.',
        'MISSING_API_KEY',
      );
    }

    const baseUrl = config?.baseUrl ?? process.env.AGENTICHUB_BASE_URL;
    if (!baseUrl) {
      throw new AgenticHubError(
        'AgenticHub: baseUrl is required. Pass it in the constructor or set AGENTICHUB_BASE_URL.',
        'MISSING_BASE_URL',
      );
    }

    this.apiKey = apiKey;
    this.baseUrl = baseUrl.replace(/\/+$/, ''); // strip trailing slash
    this.cacheTtl = config?.cacheTtl ?? DEFAULT_CACHE_TTL;
    this.maxRetries = config?.maxRetries ?? DEFAULT_MAX_RETRIES;
    this.retryInterval = config?.retryInterval ?? DEFAULT_RETRY_INTERVAL;

    // Initialise cache singleton (first call wins for maxCacheSize)
    getCache(config?.maxCacheSize ?? DEFAULT_MAX_CACHE_SIZE);
  }

  /**
   * Fetches a rendered prompt by name and alias, with optional variable substitution.
   *
   * - On cache hit (fresh): returns cached value immediately, no network call.
   * - On cache hit (stale): returns cached value immediately, fires background refresh.
   * - On cache miss: fetches from API, caches result, returns messages.
   * - API unreachable + stale entry: returns stale value, logs warning.
   * - API unreachable + cold cache: throws NETWORK_ERROR after retries.
   *
   * Note: variables are NOT part of the cache key. The same cached result is returned
   * regardless of the variables passed — caching is per prompt+alias, not per variable set.
   *
   * @param name - The prompt name (matches the `name` field in AgenticHub, not ID).
   * @param alias - The alias to resolve (e.g. 'production', 'staging').
   * @param variables - Template variables to pass to the render endpoint.
   * @returns OpenAI-compatible messages array, ready to pass to any LLM API.
   * @throws {AgenticHubError} MISSING_VARIABLES if the template requires variables not supplied.
   * @throws {AgenticHubError} API_ERROR for other non-retryable HTTP errors (401, 404, 5xx after retries).
   * @throws {AgenticHubError} NETWORK_ERROR if the API is unreachable and no stale cache entry exists.
   */
  async renderPrompt(
    name: string,
    alias: string,
    variables: Record<string, unknown> = {},
  ): Promise<Message[]> {
    const cache = getCache(DEFAULT_MAX_CACHE_SIZE);
    const cacheKey = `${this.apiKey}:${name}:${alias}`;
    const now = Date.now();

    const cached = cache.get(cacheKey);

    if (cached) {
      const age = now - cached.fetchedAt;
      if (age < this.cacheTtl) {
        // Fresh — return immediately without a network call
        return cached.value;
      }

      // Stale — return immediately and fire background refresh
      this._backgroundRefresh(name, alias, variables, cacheKey).catch((err: unknown) => {
        console.warn(
          `[AgenticHub] Background refresh failed for "${name}/${alias}" — continuing to serve stale`,
          err instanceof Error ? err.message : err,
        );
      });

      return cached.value;
    }

    // Cache miss — must fetch (blocking)
    return this._fetchAndCache(name, alias, variables, cacheKey);
  }

  /**
   * Fires a background refresh for a stale cache entry.
   * Errors are caught by the caller (renderPrompt) and logged as a warning.
   * @internal
   */
  private async _backgroundRefresh(
    name: string,
    alias: string,
    variables: Record<string, unknown>,
    cacheKey: string,
  ): Promise<void> {
    await this._fetchAndCache(name, alias, variables, cacheKey);
  }

  /**
   * Fetches from the render endpoint and stores the result in the cache.
   * @internal
   */
  private async _fetchAndCache(
    name: string,
    alias: string,
    variables: Record<string, unknown>,
    cacheKey: string,
  ): Promise<Message[]> {
    const url = `${this.baseUrl}/prompts/${encodeURIComponent(name)}/${encodeURIComponent(alias)}/render`;

    let response: Response;
    try {
      response = await fetchWithRetry(
        url,
        {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${this.apiKey}`,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({ variables }),
        },
        this.maxRetries,
        this.retryInterval,
      );
    } catch (err) {
      // fetchWithRetry threw — all retry attempts exhausted at the network level
      throw new AgenticHubError(
        `AgenticHub: network error fetching "${name}/${alias}" — ${err instanceof Error ? err.message : String(err)}`,
        'NETWORK_ERROR',
      );
    }

    if (!response.ok) {
      const body = await response.json().catch(() => undefined);

      // 400 with missing variables — special case
      if (
        response.status === 400 &&
        body &&
        typeof body === 'object' &&
        'missing' in body &&
        Array.isArray((body as Record<string, unknown>).missing)
      ) {
        throw new AgenticHubError(
          `AgenticHub: missing required template variables: ${(body as { missing: string[] }).missing.join(', ')}`,
          'MISSING_VARIABLES',
          400,
          body,
        );
      }

      throw new AgenticHubError(
        `AgenticHub API error ${response.status} for "${name}/${alias}"`,
        'API_ERROR',
        response.status,
        body,
      );
    }

    const data = await response.json() as { messages: Message[] };
    const messages = data.messages;

    const cache = getCache(DEFAULT_MAX_CACHE_SIZE);
    cache.set(cacheKey, { value: messages, fetchedAt: Date.now() });

    return messages;
  }
}

# @agentichub/sdk

TypeScript SDK for [AgenticHub](https://github.com/talhaanwarch/agenticprompthub). Fetch rendered prompts at runtime without touching your database or managing version numbers.

## Installation

```bash
npm install @agentichub/sdk
```

## Quickstart

```typescript
import AgenticHub from '@agentichub/sdk';
import OpenAI from 'openai';

const hub = new AgenticHub({
  apiKey: process.env.AGENTICHUB_API_KEY,
  baseUrl: process.env.AGENTICHUB_BASE_URL,
});

const openai = new OpenAI();

async function runAgent(userMessage: string) {
  // Fetch the latest production prompt — no deploy needed to update it
  const messages = await hub.renderPrompt('summarise-article', 'production', {
    article: userMessage,
  });

  const response = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages,
  });

  return response.choices[0].message.content;
}
```

## Configuration

| Option | Environment Variable | Default | Description |
|--------|---------------------|---------|-------------|
| `apiKey` | `AGENTICHUB_API_KEY` | **required** | Your AgenticHub API key |
| `baseUrl` | `AGENTICHUB_BASE_URL` | **required** | AgenticHub API base URL (e.g. `http://localhost:3000/api/v1`) |
| `cacheTtl` | — | `60000` (60s) | Milliseconds before a cached result is considered stale |
| `maxCacheSize` | — | `500` | Maximum number of prompt entries in the in-process cache |
| `maxRetries` | — | `1` | Retry attempts on transient failure (2 total attempts) |
| `retryInterval` | — | `500` | Milliseconds between retries |

**Create one instance at process startup and reuse it.** The cache is a module-level singleton; `maxCacheSize` is set by the first constructor call.

## Error Handling

```typescript
import { AgenticHubError } from '@agentichub/sdk';

try {
  const messages = await hub.renderPrompt('my-prompt', 'production', vars);
} catch (err) {
  if (err instanceof AgenticHubError) {
    switch (err.code) {
      case 'MISSING_VARIABLES':
        // Template requires variables you didn't supply
        console.error('Missing template variables:', (err.body as { missing: string[] }).missing);
        break;
      case 'NETWORK_ERROR':
        // API unreachable — no stale cache entry available
        console.error('AgenticHub API unreachable. Check AGENTICHUB_BASE_URL.');
        break;
      case 'API_ERROR':
        // Non-retryable HTTP error (401 invalid key, 404 prompt not found, etc.)
        console.error(`AgenticHub API error ${err.statusCode}`);
        break;
    }
  }
  throw err;
}
```

## Caching Behaviour

The SDK maintains an in-process LRU cache:

- **Cache key:** `${apiKey}:${promptName}:${alias}` — scoped per team, per prompt, per alias
- **Variables are NOT cached separately.** The cache stores the rendered output for a prompt+alias combination. Different variable values on the same prompt+alias share the same cache slot. If variable-sensitive caching is needed, that is a future enhancement.
- **Stale-while-revalidate:** On a stale hit, the cached value is returned immediately and a background refresh is triggered. This keeps latency predictable even when the TTL expires.
- **API unreachable + stale entry:** The stale value is served silently and a warning is logged: `[AgenticHub] Background refresh failed for "name/alias" — continuing to serve stale`.
- **API unreachable + cold cache:** The SDK throws `AgenticHubError` with `code: 'NETWORK_ERROR'` after exhausting retries.

## Requirements

- Node.js 18+ (uses native `fetch`)
- No browser support (LLM calls are server-side)
